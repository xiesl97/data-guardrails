# 数据工程护栏

Data Guardrails 是一套面向高风险数据工程和科研数据流程的通用 Agent
Skill。

它要求 AI Agent 透明地处理数据，保留用户对重要决策的知情权和裁量权，复用权威工具，
验证数据产品，并留下可审计的工作交接。它不会规定任何项目专属的算法、仪器、目录结构或
固定处理阶段。

[English README](README.md)

## 核心原则

- 禁止静默修改、排除、修复、替换或忽略数据。
- 对影响数据含义和下游结论的重要决策，用户必须保持知情和控制。
- 优先复用项目和领域内的权威实现，不擅自重复造轮子。
- 保留源数据、来源记录、验证证据和已知限制。
- 必要决策尚未明确或数据产品可能无效时，必须停止并汇报。
- 将产生数据的处理步骤与下游解释和分析区分开。

## 目录结构

```text
data-guardrails/
├── SKILL.md
└── references/
    ├── audit-checklist.md
    └── decision-gates.md
```

- `SKILL.md`：核心行为准则和触发描述。
- `references/audit-checklist.md`：可根据具体项目选用的审计清单。
- `references/decision-gates.md`：规定 Agent 何时可以继续、何时必须请求用户裁决、
  何时必须停止。

`references` 是通用 skill 的组成部分，存放由 `SKILL.md` 按需引用的详细规则，
不是任何特定 Agent 平台的配置。

## 安装

将本仓库交给准备使用它的 AI Agent，并要求该 Agent 将其安装为可复用 skill。

例如：

```text
请将这个仓库安装为你所在 Agent 环境中的通用 skill：
<仓库路径或 URL>
```

Agent 应在安装前检查自身支持的 skill 格式和能力。如果能够读取引用资料，应保留
`references` 目录；如果只能使用单个指令文件，应将必要的引用内容合并到安装后的 skill
中，同时保持规则原意不变。

## 使用方式

开展重要数据工作时，可以显式调用：

```text
使用 $data-guardrails 构建并验证这个数据处理流程。
```

当任务符合 `SKILL.md` 中的触发描述时，支持自动发现的 Agent 也可能自动启用它。

本 skill 应与具体项目的数据处理标准配合使用。项目标准负责定义实际数据来源、批准使用的
算法、参数、质量判据和输出要求；Data Guardrails 负责约束 Agent 如何执行、验证、记录
和汇报这些决定。

## 适用范围

适用于：

- 数据获取、筛选、转换、清洗、校准和导出
- 科研和科学数据裁剪
- 分析就绪数据集的生成
- 多步骤数据处理流水线
- 静默降级或擅自处理可能导致结果失效的工作

它不能替代领域知识、权威处理软件、项目标准或用户批准。
